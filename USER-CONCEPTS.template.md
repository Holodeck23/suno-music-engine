# User Concept Library (Template)

This is your personal scratch space — copy this file to `USER-CONCEPTS.md` and fill it in. `USER-CONCEPTS.md` is gitignored so your concepts stay local.

The reusable methodology lives in `SKILL.md` and `references/`. Keep your personal concept seeds, worked examples, and album thematic frameworks here so the methodology stays portable for anyone else.

---

## How to Use This File

Three kinds of content live here:

1. **Concept Seeds** — Pre-developed thematic frameworks for albums you might one day write. A concept seed is a thesis + a 3-act sonic structure + thematic threads. The skill's album mode will pull from these when you say "let's plan an album."
2. **Worked Examples** — Tracks or albums you've actually built using this skill. Useful as concrete reference material for "what does a fully-considered prompt look like in practice."
3. **Personal Mappings** — If you use the personal-context-translation mode, save the instrument-to-meaning mappings for your own life here so they stay consistent across tracks.

Delete any sections you don't need. Add new ones freely.


**Style Prompt Breakdown:** Walk through each phrase. Why is it there? What does it earn? What would the track lose if you removed it?

**Section-by-Section Architecture:** For each section in the metatag/lyrics structure, explain the compositional intent. Not "what it sounds like" — "why I chose this." (See `USER-CONCEPTS.md` SEAM example in the original install for reference format.)
